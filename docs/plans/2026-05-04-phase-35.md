# Phase 3.5 — Camp Status, QR Detail, UI Rename

**Goal:** Add open/close toggle to admin camp detail, show a "Camp Closed" screen in the parent flow, surface the QR code on the session detail page, and rename "session" to "camp" in all UI labels.

**Architecture:** One new DB column (`is_open`) on `camp_sessions`. API layer gets one new function (`toggleCampStatus`) and one new lightweight function (`getCampStatus` — used by parent flow to check open/closed before rendering). Admin detail page gains a status badge + toggle button + QR section. `DreamFlow` gains a new `camp-closed` step shown when `is_open = false`.

**Design Patterns:** Extend existing patterns — no new abstractions. Status check in parent flow uses same Supabase client. Toggle is optimistic UI with rollback on error.

**Tech Stack:** Supabase SQL migration, TypeScript, React, Tailwind, existing `qrcode` npm package already installed.

**Spec:** `public/prototype_design_system.html` (approved 2026-05-04) — Camp Detail tab and Parent Closed screen tabs.

### Chunk 2.2 — Add `toggleCampStatus` + `getCampStatus`

**Files:** Modify `src/lib/api.ts` — add two new exported functions after `markCardGenerated`.

```typescript
export async function toggleCampStatus(id: string, isOpen: boolean): Promise<void> {
  const { error } = await supabase
    .from("camp_sessions")
    .update({ is_open: isOpen })
    .eq("id", id);
  if (error) throw error;
}

export async function getCampStatus(id: string): Promise<{ is_open: boolean }> {
  const { data, error } = await supabase
    .from("camp_sessions")
    .select("is_open")
    .eq("id", id)
    .single();
  if (error) throw error;
  return { is_open: data.is_open };
}
```

**Verify:** `npm run build` passes.

**Commit:** `feat(api): add toggleCampStatus and getCampStatus`

**Step 2 — Add `isOpen` state + toggle handler in `SessionDetail` component:**
```typescript
const [isOpen, setIsOpen] = useState<boolean | null>(null);
const [toggling, setToggling] = useState(false);

// Inside the .then() callback where session is set:
setIsOpen(session.is_open);

const handleToggle = async () => {
  if (isOpen === null) return;
  const next = !isOpen;
  setIsOpen(next); // optimistic
  setToggling(true);
  try {
    await toggleCampStatus(sessionId, next);
  } catch {
    setIsOpen(!next); // rollback
  } finally {
    setToggling(false);
  }
};
```

**Step 3 — Add badge + button below the `<h1>`:**
```tsx
<div className="flex items-center gap-3 mb-6">
  <span className={`inline-flex items-center px-3 py-1 rounded-full text-xs font-bold uppercase tracking-wider ${
    isOpen
      ? "bg-emerald-900/40 text-emerald-400"
      : "bg-red-900/40 text-red-400"
  }`}>
    {isOpen ? "Open" : "Closed"}
  </span>
  <button
    onClick={handleToggle}
    disabled={toggling || isOpen === null}
    className={`px-3 py-1.5 rounded-lg text-xs font-semibold border-2 disabled:opacity-40 transition ${
      isOpen
        ? "border-red-400 text-red-400 hover:bg-red-400/10"
        : "border-emerald-400 text-emerald-400 hover:bg-emerald-400/10"
    }`}
  >
    {toggling ? "Saving…" : isOpen ? "Close Camp" : "Reopen Camp"}

</div>
```
